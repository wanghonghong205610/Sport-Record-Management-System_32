# Sport-Record-Management-System_32
运动记录系统（开源文化运动期末考察报告）
项目简介
Sport-Record-Management-System 是一款基于 Java 与 MySQL 数据库技术开发的轻量型运动记录管理系统。系统聚焦个人运动数据的数字化管理，提供运动记录添加、多维度查询、数据统计及记录维护等核心功能，以简洁的控制台交互替代传统纸质 / Excel 记录方式，帮助用户便捷追踪运动轨迹、统计运动时长，助力养成规律运动习惯。
项目背景
在日常运动管理中，传统记录方式（如手写笔记、Excel 表格）存在查询不便、统计繁琐、数据易丢失等问题，难以满足用户快速追溯运动历史、分析运动规律的需求。本项目通过整合 Java 业务逻辑处理与 MySQL 数据持久化能力，构建轻量化、易操作的运动记录平台，实现运动数据的规范化存储、高效查询与简单统计，降低个人运动管理成本。
技术栈
编程语言
Java：负责核心业务逻辑实现（CRUD 操作）、控制台交互界面开发、数据格式处理
数据库
MySQL 5.7+：存储运动记录核心数据（运动类型、时长、日期、备注等）
开发工具与环境
IDE：IntelliJ IDEA
数据库管理工具：Navicat / MySQL Workbench
JDK 版本：JDK 8 及以上
依赖：MySQL Connector/J（Java 连接 MySQL 驱动）
功能模块
1. 运动记录管理模块
记录添加：用户输入运动类型（如跑步、游泳）、运动时长（分钟）、运动日期（yyyy-MM-dd）及备注（可选），系统将数据持久化至数据库
记录删除：用户通过记录 ID 定位目标记录，执行删除操作（删除前展示所有记录供参考）
2. 查询统计模块
全量查询：按运动日期倒序展示所有运动记录，包含记录 ID、运动类型、时长、日期及备注信息
条件查询：支持按指定日期查询该日所有运动记录，快速定位特定时段运动数据
简单统计：自动计算总运动次数、累计运动时长、平均每次运动时长，直观呈现运动概况
3. 数据维护模块
输入校验：对运动类型（非空）、运动时长（正整数）、日期格式（yyyy-MM-dd）进行合法性校验，避免无效数据录入
资源管理：自动管理数据库连接资源，确保连接创建与关闭规范，避免资源泄露
