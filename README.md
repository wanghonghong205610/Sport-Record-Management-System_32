# Sport-Record-Management-System_32
单人运动记录系统，提供运动记录添加、多维度查询、数据统计及记录维护等核心功能（开源文化运动期末考察报告）

## 项目简介

Sport-Record-Management-System 是一款基于 Java 与 MySQL 数据库技术开发的轻量型运动记录管理系统。系统聚焦个人运动数据的数字化管理，提供运动记录添加、多维度查询、数据统计及记录维护等核心功能，以简洁的控制台交互替代传统纸质 / Excel 记录方式，帮助用户便捷追踪运动轨迹、统计运动时长，助力养成规律运动习惯。

## 项目背景

在日常运动管理中，传统记录方式（如手写笔记、Excel 表格）存在查询不便、统计繁琐、数据易丢失等问题，难以满足用户快速追溯运动历史、分析运动规律的需求。本项目通过整合 Java 业务逻辑处理与 MySQL 数据持久化能力，构建轻量化、易操作的运动记录平台，实现运动数据的规范化存储、高效查询与简单统计，降低个人运动管理成本。

## 功能模块

1. 运动记录管理模块
记录添加：用户输入运动类型（如跑步、游泳）、运动时长（分钟）、运动日期（yyyy-MM-dd）及备注（可选），系统将数据持久化至数据库
记录删除：用户通过记录 ID 定位目标记录，执行删除操作（删除前展示所有记录供参考）

2. 查询统计模块
全量查询：按运动日期倒序展示所有运动记录，包含记录 ID、运动类型、时长、日期及备注信息
条件查询：支持按指定日期查询该日所有运动记录，快速定位特定时段运动数据
简单统计：自动计算总运动次数、累计运动时长、平均每次运动时长，直观呈现运动概况

3. 数据维护模块
输入校验：对运动类型（非空）、运动时长（正整数）、日期格式（yyyy-MM-dd）进行合法性校验，避免无效数据录入
资源管理：自动管理数据库连接资源，确保连接创建与关闭规范，避免资源泄露

## Mysql数据库

### 运动记录表（sport_records）


记录唯一ID	  INT	 PRIMARY KEY， AUTO_INCREMENT，  NOT NULL	  用于定位和删除记录

运动类型	  VARCHAR(50)，  NOT  NULL	  如：跑步、游泳、跳绳、篮球

运动时长（分钟）    INT， NOT  NULL，  CHECK(duration > 0)	   必须为正整数

运动日期	   DATE，	NOT NULL	 格式：yyyy-MM-dd

运动备注	   VARCHAR(200)，	DEFAULT 	  可选，如：晨跑、操场、配速 6 分钟 / 公里

记录创建时间	   TIMESTAMP， 	DEFAULT，  CURRENT_TIMESTAMP	   自动填充，无需手动输入

## 运行程序 

确保 MySQL 服务处于运行状态

定位到 app/SportRecordApp.java，运行 main 方法启动控制台程序

功能操作：
<br>1.输入数字 1-4 选择对应功能（添加 / 查询所有 / 按日期查询 / 删除记录）
<br>2.按提示输入相关信息（如运动类型、日期需符合 yyyy-MM-dd 格式）
<br>3.输入 0 退出程序

## 注意事项
1.数据库连接失败排查：检查 MySQL 服务是否启动、用户名 / 密码是否正确、驱动 Jar 包是否导入成功

2.输入格式要求：运动时长需为正整数，日期需严格遵循 yyyy-MM-dd 格式（如 2025-11-30）

3.数据安全：建议定期备份 sport_db 数据库，避免误操作导致数据丢失

4.兼容性：JDK 版本建议不低于 8，MySQL 版本不低于 5.7，避免因版本不兼容出现异常
